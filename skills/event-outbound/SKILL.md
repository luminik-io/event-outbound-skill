# Event Outbound Skill

Built for AEs, SDRs, and event marketers working B2B trade shows, conferences, and industry events.

- **AEs / SDRs**: target the attendee list of a conference your company is exhibiting at or attending. Generate pre-event, day-of, and post-event touches that actually get opened.
- **Event marketers**: run dinner invites, booth-visit nudges, speaker meet-and-greet outreach, and side-event marketing at large trade shows.
- **Founders doing their own outbound**: same workflow, same validated output.

Grounded in 20k+ personalised emails and LinkedIn messages across 50+ trade shows, conferences, and industry events. $6M+ in sourced pipeline across fintech (identity verification), cybersecurity, and B2B SaaS. This plugin is the validated version of what actually worked.

## Purpose

Every cold-email generator claims "proven frameworks." This one validates every touch against hard rules before it lands: subject ≤4 words, body 50–100 words, banned-words blocklist, pronoun ratio favouring the reader, CTA pattern favouring concrete offers over meeting-asks. The generator is commodity; the validator is the product.

The `event-outbound` skill automates the creation of targeted outreach cadences and enforces tight messaging constraints so the output reads like someone who has actually worked the job, not a generic LLM blast.

## When to Use

Use this skill when the user needs to create an outreach sequence for an upcoming B2B event, targeting specific company personas. It is designed for scenarios where detailed personalisation and strict adherence to validated messaging rules are both required. It is not for generic always-on outbound, lifecycle email, or content marketing.

## How to Use the Skill

To generate an outreach sequence, provide the following inputs:

1.  **Event Context (`EventContext`)**: Details about the event, including name, dates, location, agenda, speakers, and exhibitors. This information is typically extracted using the `event-scraper` agent.
2.  **Company ICP (`CompanyICP`)**: Firmographic data about the target company, crucially including an array of `AttendeePersona` objects. Each persona outlines roles, seniority, priorities, and pain points. This is typically generated by the `icp-analyser` agent.
3.  **Sequence Parameters (`SequenceParams`)**: Configuration for the outreach, such as `leadTimeWeeks` (how far in advance to start outreach), `channels` (email or LinkedIn), and `sendingIdentity` (the sender's name, title, and company).

The skill will then utilise the `sequencer` agent to generate a `SequencerOutput` containing one `OutreachSequence` per `AttendeePersona` defined in the `CompanyICP`. Each `OutreachSequence` will consist of multiple `OutreachTouch` messages, timed according to the `leadTimeWeeks` and distributed across the specified `channels`.

Each generated `OutreachTouch` is automatically validated against strict cold-outreach benchmarks (subject line length, body word count, pronoun ratio, CTA type, banned words) derived from the `cold-email-benchmarks.json` data. This ensures high-quality, compliant output.

### Core Agents

*   **`event-scraper.ts`**: Extracts event details from a URL.
*   **`icp-analyser.ts`**: Analyses a company URL to populate `CompanyICP` with multiple `AttendeePersona` objects.
*   **`persona-analyser.ts`**: Extracts details from a LinkedIn profile or provided text to create an `AttendeePersona`.
*   **`sequencer.ts`**: The main agent responsible for generating the `OutreachSequence` based on the provided `EventContext`, `CompanyICP`, and `SequenceParams`.
*   **`lib/timeline.ts`**: Provides pure timing logic for distributing touches across the lead time.

### Data Grounding

The `sequencer.ts` agent is grounded by internal reference files in the `data/` directory. These files encode generic cold-outbound craft principles as pattern shapes and hard-filter validation rules. None of these internal sources are ever cited in the generated output.

### Output Structure

The skill returns a `SequencerOutput` object, which contains `sequencesByPersona`. This object is a dictionary where each key is a `personaId` and the value is the corresponding `OutreachSequence` for that persona. Each persona gets a distinct sequence.
